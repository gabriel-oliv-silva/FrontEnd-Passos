O CSS (Cascading Style Sheets) é o responsável por dar vida e beleza ao HTML. Se o HTML é o esqueleto de uma casa, o CSS é a pintura, a decoração e o design de interiores.

Existem 3 formas de conectar o CSS ao seu HTML:

  • Inline: Direto na tag HTML usando o atributo style="...". (Não recomendado para projetos grandes).
  • Interno: Dentro da tag <style> no <head> do documento.
  • Externo: Em um arquivo separado (ex: style.css) lincado através da tag <link rel="stylesheet" href="style.css">. Esta é a melhor prática!
Atenção: O termo "Cascading" (Cascata) significa que o CSS lê o código de cima para baixo. Se você der a cor azul para um botão e, na linha de baixo, der a cor vermelha para o mesmo botão, ele ficará vermelho.

Para o CSS saber qual elemento ele deve pintar, precisamos selecioná-lo. Usamos Seletores para isso.

  • Seletor de Tag: Seleciona pelo nome do elemento HTML. Ex: p { ... }
  • Seletor de Classe (Class): Usa-se um ponto (.) antes do nome. Pode ser repetido várias vezes na página. Ex: .botao-azul { ... }
  • Seletor de ID: Usa-se a cerquilha (#). Deve ser único na página. Ex: #cabecalho-principal { ... }

Especificidade: É um sistema de pontos que o navegador usa para resolver conflitos de estilo. A regra é: Inline > ID > Classe > Tag. O ID "vence" a Classe, e a Classe "vence" a Tag.

                            CSS
/* Tag - Especificidade Baixa */
h2 { color: black; }

/* Classe - Especificidade Média */
.titulo-destaque { color: blue; }

/* ID - Especificidade Alta */
#titulo-unico { color: red; }

Na web, tudo é uma caixa (retângulo). O comportamento dessas caixas é ditado pelo Box Model (Modelo de Caixa). Ele é composto por quatro áreas:

  • Content (Conteúdo): Onde o texto ou imagem fica. Definido por width e height.
  • Padding (Preenchimento): Espaço interno, entre o conteúdo e a borda.
  • Border (Borda): A linha que envolve o preenchimento e o conteúdo.
  • Margin (Margem): Espaço externo, afasta a sua caixa das outras caixas ao redor.

Unidades de Medida Principais:

Além de Pixels (px), usamos unidades relativas como % (porcentagem do pai), rem (relativo ao tamanho da fonte raiz), vh e vw (relativo à altura e largura da tela visualizada).

O Flexbox (Flexible Box Layout) é um modelo focado em organizar elementos em uma única direção (linha ou coluna). É perfeito para alinhar botões, criar menus de navegação e centralizar coisas (o que antigamente era um pesadelo!).

Para ativá-lo, você coloca display: flex; no elemento PAI. Feito isso, as principais propriedades são:

  • flex-direction: Define se os itens ficam em linha (row) ou em coluna (column).
  • justify-content: Alinha os itens no eixo principal (Ex: center, space-between).
  • align-items: Alinha os itens no eixo cruzado (ex: centralizar verticalmente).
                            CSS
.container-flex {
  display: flex;
  justify-content: flex-start;
  align-items: center;
  gap: 15px; /* Espaço entre os itens */
}

Enquanto o Flexbox é excelente para uma direção, o CSS Grid é o rei do layout Bidimensional (linhas e colunas ao mesmo tempo). Ele é ideal para montar a estrutura principal inteira do seu site.

Com o display: grid;, você desenha uma "grade" e diz exatamente onde cada elemento deve ficar.

  • grid-template-columns: Define a quantidade e a largura das colunas.
  • grid-template-rows: Define a altura das linhas.
  • A unidade fr (fração) é exclusiva do Grid. 1fr 1fr 1fr cria 3 colunas de tamanhos perfeitamente iguais.
                            CSS
.grid-container {
  display: grid;
  /* Cria 3 colunas de tamanhos iguais */
  grid-template-columns: repeat(3, 1fr);
  gap: 10px; /* Espaçamento entre os blocos */
}

Hoje, a maioria das pessoas acessa a internet pelo celular. O seu site precisa se adaptar a qualquer tamanho de tela. Fazemos isso usando Media Queries.

Mobile-First é a prática de escrever o CSS base para a tela do celular e, usando Media Queries, ir alterando o layout conforme a tela vai crescendo (para tablets e PCs).

                            CSS
/* CSS Base (Celulares) */
.coluna { width: 100%; }

/* Se a tela for MAIOR que 768px (Tablets e PCs) */
@media (min-width: 768px) {
  .coluna { width: 50%; } /* Agora ficam lado a lado */
}
Atenção: Sempre teste seu site diminuindo a janela do navegador no computador ou inspecionando pelo "Modo de Dispositivo" do Google Chrome.

Movimento atrai a atenção. O CSS possui ferramentas poderosas e leves para isso sem precisar de JavaScript.

  • Transitions (Transições): Servem para suavizar mudanças de estado. Se você muda a cor no :hover (quando o mouse passa por cima), a transição faz essa mudança ser fluida em vez de seca.
  • Animations (@keyframes): Para movimentos contínuos e complexos. Você cria uma "linha do tempo" (de 0% a 100%) e diz o que o elemento faz em cada passo.
                            CSS
/* Transição simples de botão */
.botao { background: blue; transition: background 0.3s ease; }
.botao:hover { background: darkblue; }

/* Animação com Keyframes */
@keyframes giro {
  0%   { transform: rotate(0deg); }
  100% { transform: rotate(360deg); }
}
.rotacao { animation: giro 2s linear infinite; }

Projetos grandes requerem organização. As Custom Properties (Variáveis CSS) permitem que você guarde uma cor ou tamanho uma única vez e o chame pelo nome no resto do código. Se o cliente pedir para mudar a cor do site de azul para roxo, você só muda na variável!

Geralmente, declaramos as variáveis na pseudo-classe :root para que todo o site tenha acesso a elas.

                            CSS
:root {
  --cor-primaria: #264de4;
  --espacamento-padrao: 15px;
}

.cartao {
  background-color: var(--cor-primaria);
  padding: var(--espacamento-padrao);
}

Recursos Visuais Modernos:

  • linear-gradient: Cria degrades de cores no background.
  • box-shadow: Cria sombras suaves ao redor dos elementos para dar profundidade e efeito de "card" flutuando.
  • filter: Aplica efeitos fotográficos a elementos (como blur(), grayscale()).
