A Máquina do Tempo dos Desenvolvedores

O Git é um sistema de controle de versão. Ele rastreia as mudanças feitas em arquivos ao longo do tempo, permitindo que você volte para versões anteriores e trabalhe em equipe sem que um sobrescreva o trabalho do outro.

Analogia:

Pense no Git como o "Save Point" de um videogame. Se você cometer um erro terrível na fase atual, basta carregar o jogo no ponto seguro anterior e tentar de novo.

Atenção

Git não é GitHub! O Git é a ferramenta (o motor) instalada no seu computador. O GitHub é um site (a garagem) onde você guarda seus projetos usando o Git.

Trazendo o Git para sua máquina

Para usar o Git, precisamos instalá-lá no nosso computador. O processo varia um pouco de acordo com o Sistema Operacional:

  • Windows: Baixe o instalador no site oficial (git-scm.com) e siga o "Next, Next, Install". Ele instala o Git Bash, o terminal que usaremos.
  • Mac: Se você abrir o terminal e digitar git, o Mac geralmente sugere a instalação. Ou use o Homebrew: brew install git.
  • Linux: Use o gerenciador do sistema. Ex: sudo apt install git (Ubuntu/Debian).
# Para verificar se a instalação deu certo, digite no terminal:
git --version
git version 2.40.1.windows.1

Apresente-se ao Git

Antes de começar a salvar as versões do seu código, o Git precisa saber quem é você. Todo registro (commit) que você fizer carregará seu nome e email.

Analogia:

É como colocar um crachá de identificação antes de entrar na empresa. Tudo o que você fizer lá dentro ficará registrado no sistema com o seu nome.

# Configure seu nome:
git config --global user.name "Seu Nome Aqui"
# Configure seu email:
git config --global user.email "seu-email@exemplo.com"
Dica

A flag --global significa que essas configurações valerão para todos os projetos no seu computador. Você só precisa fazer isso uma vez!

Começando a Mágica

Para o Git começar a vigiar uma pasta, você precisa transformá-la em um repositório. A partir daí, o fluxo de trabalho tem duas etapas fundamentais: separar o que vai ser salvo (add) e registrar oficialmente (commit).

Analogia da Caixa de Mudança:
  • git init: Comprar a caixa vazia.
  • git add: Colocar os itens dentro da caixa (preparação).
  • git commit: Fechar a caixa com fita, colocar uma etiqueta (mensagem) e guardar.
# Inicia o repositório na pasta atual:
git init
Initialized empty Git repository in /caminho/do/projeto/.git/ # Adiciona o arquivo index.html "na caixa":
git add index.html
# Salva a versão com uma mensagem:
git commit -m "Cria a página inicial"
[main (root-commit) 4a2f8c] Cria a página inicial

Investigando o Passado

Depois de realizar vários commits, você pode querer ver o que já foi feito, ou quem fez determinada alteração. O Git fornece ferramentas poderosas para auditar o histórico.

Inspecionando
  • git log: Lista o histórico de commits.
  • git status: Mostra o estado atual (se há arquivos modificados ou esperando o commit).
git log --oneline
4a2f8c9 (HEAD -> main) Corrige o erro no botão de login
9b1e7f2 Adiciona CSS a página de contatos
d83c1a4 Cria a página inicial

Trabalhando em Universos Paralelos

No Git, a linha principal do seu código costuma se chamar main. Se você quer testar uma ideia nova ou criar uma funcionalidade sem arriscar quebrar o código principal, você cria uma branch (ramificação).

Analogia:

É como o "Multiverso" nos filmes. Você cria uma linha do tempo alternativa. Lá você pode explodir coisas à vontade. Se der certo, você traz o resultado de volta para o universo original (processo chamado Merge).

# Cria e muda para uma nova branch chamada "nova-tela":
git checkout -b nova-tela
Switched to a new branch 'nova-tela' # Para juntar as coisas depois, você volta para a main:
git checkout main
# E puxa as alterações:
git merge nova-tela

A Nuvem do Desenvolvedor

Até agora, tudo estava apenas no seu computador. Mas e se o seu PC quebrar? Ou se você precisar trabalhar com alguém do outro lado do mundo? É aí que entra o GitHub.

O GitHub é uma plataforma online que hospeda repositórios Git na nuvem.

Conectando os mundos

Para enviar seu código local para o GitHub, você precisa apontar para qual endereço ele deve ir. Nós chamamos esse endereço remoto de origin por convenção.

# Conectando seu repositório local com o do GitHub:
git remote add origin https://github.com/seu-usuario/seu-projeto.git

Subindo e Descendo Código

Depois de conectar seu repositório local ao GitHub, você precisa enviar os commits para lá (Upload) ou baixar as atualizações que outras pessoas fizeram (Download).

Analogia:
  • git push (Empurrar): Enviar o seu novo vídeo para o YouTube.
  • git pull (Puxar): Atualizar o feed do seu celular para ver os posts mais novos.
  • git clone (Clonar): Fazer o download completo de uma série da Netflix pela primeira vez.
# Enviando código para o GitHub (branch main):
git push origin main

Trabalhando como um Profissional

O Git é livre e aceita qualquer coisa, mas trabalhar em equipe exige etiqueta. Escrever mensagens de commit ruins ou enviar códigos quebrados atrapalha muito.

O que fazer e o que NÃO fazer

Ruim: git commit -m "consertei umas coisas ai"

Bom: git commit -m "Corrige bug no alinhamento do cabeçalho mobile"

  • Commits Atômicos: Faça commits pequenos e focados. Não altere 30 arquivos de 5 tarefas diferentes e agrupe em um único commit.
  • Escreva no Imperativo: "Adiciona tela de login" ao invés de "Adicionado" ou "Adicionando".
  • Sempre faça Pull: Antes de começar a programar no dia, faça um `git pull` para garantir que seu código não está desatualizado.

Seu Site no Ar de Graça

O GitHub Pages é um serviço de hospedagem de site estático gratuito embutido no GitHub. Se você tem um projeto com HTML, CSS e JavaScript no seu repositório, o GitHub gera um link público para ele!

Analogia:

É como se o GitHub transformasse a caixa onde você guardou o código em uma vitrine de loja, para que qualquer pessoa com a URL possa ver o site funcionando, em vez de apenas ler os códigos.

Como ativar:
  1. Faça o push do seu projeto HTML para o GitHub.
  2. Vá na aba Settings do repositório.
  3. No menu lateral, clique em Pages.
  4. Em Source, selecione a branch main e clique em Save.
  5. Aguarde 1 minutinho e seu link estará pronto! Ex: https://seu-usuario.github.io/seu-projeto/
