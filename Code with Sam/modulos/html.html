A internet funciona como um restaurante: você (o cliente) faz um pedido pelo navegador, e o servidor (o cozinheiro) te devolve a página pronta. O HTML (HyperText Markup Language) não é uma linguagem de programação, mas sim uma linguagem de marcação.

Ele é responsável por montar o "esqueleto" da página. Todo arquivo HTML precisa de uma estrutura básica para que o navegador entenda o que está lendo.

Atenção: A tag <!DOCTYPE html> sempre deve ser a primeira linha do seu código. Ela avisa ao navegador que você está usando a versão mais recente do HTML (HTML5).
  • <html>: A tag raiz que abraça todo o site.
  • <head>: A "cabeça". Fica invisível para o usuário. Guarda configurações, título da aba e links de CSS.
  • <body>: O "corpo". Tudo que o usuário vê na tela (textos, imagens, botões) fica aqui dentro.
                            HTML
<!DOCTYPE html> 
 <html lang="pt-BR"> 
  <head> 
    <meta charset="UTF-8">
    <title>Minha Primeira Página</title>
  </head>
  <body>
    Olá, Mundo! Este é o corpo do site.
  </body>
</html>
Resultado ao vivo

Olá, Mundo! Este é o corpo do site.

O HTML tem 6 níveis de títulos: de <h1> (mais importante) até <h6> (menos importante). Isso cria uma hierarquia que ajuda tanto o leitor quanto o Google a entender sua página. Para textos comuns, usamos a tag de parágrafo <p>.

Regra de Ouro: Use apenas um H1 por página. Os H2 são subtópicos do H1, H3 são subtópicos dos H2, e assim por diante.

Tags de formatação de texto:

  • <strong> → Negrito com importância semântica.
  • <em> → Itálico com ênfase.
  • <mark> → Texto destacado (marcador amarelo).
  • <del> → Texto riscado (deletado).
  • <sup> e <sub> → Sobrescrito (ex: x²) e Subscrito (ex: H₂O).
                            HTML
<h1>Título Principal (só um por página)</h1> 
<h2>Subtítulo</h2> 
<h3>Sub-subtítulo</h3>

<p>Parágrafo comum com <strong>negrito</strong> e <em>itálico</em>.</p>
<p>A fórmula da água é H<sub>2</sub>O e a equação é E=mc<sup>2</sup></p>
<p>Preço: <del>R$100</del> <strong>R$79</strong></p>

Sem links, a internet seria apenas um documento gigante. Para criar links (âncoras), usamos a tag <a>. Para exibir imagens, usamos a tag <img>, que é uma "tag vazia" (não precisa de fechamento).

Acessibilidade: Nunca esqueça o atributo alt nas imagens. Ele descreve a imagem para deficientes visuais e é exibido caso a imagem quebre.
  • href: Atributo do link que diz para qual URL a pessoa vai.
  • target="_blank": Atributo que faz o link abrir em uma nova aba.
  • src: Atributo da imagem (Source). O caminho ou URL de onde a foto está vindo.
HTML <!-- Link abrindo em nova aba -->
<a href="https://google.com" target="_blank">Acessar o Google</a>

<!-- Imagem com descrição -->
<img src="https://via.placeholder.com/150" alt="Quadrado cinza de exemplo">

Para agrupar itens em tópicos, o HTML oferece dois tipos principais de listas. Já as tabelas são usadas estritamente para exibir dados tabulares, e não para criar o layout do site (isso era feito nos anos 90!).

  • <ul> (Unordered List): Lista com marcadores (bolinhas).
  • <ol> (Ordered List): Lista numerada (1, 2, 3...).
  • <li> (List Item): Os itens dentro das listas.

Para Tabelas:

  • <table> → Cria a tabela.
  • <tr> (Table Row) → Cria uma nova linha horizontal.
  • <th> (Table Header) → Célula de cabeçalho (fica em negrito).
  • <td> (Table Data) → Célula de dado comum. 
                            HTML
<!-- Lista Ordenada -->
<h4>Top 3 Frutas</h4>
<ol>
  <li>Maçã</li>
  <li>Banana</li>
  <li>Uva</li>
</ol>

<!-- Tabela -->
<table border="1">
  <tr>
    <th>Produto</th>
    <th>Preço</th>
  </tr>
  <tr>
    <td>Teclado</td>
    <td>R$ 150</td>
  </tr>
</table>

Os formulários são a principal forma de receber dados dos usuários (login, cadastro, mensagens). Tudo começa envolvendo os campos na tag <form>.

Dica de UX: Sempre use a tag <label> atrelada ao id do <input>. Isso permite que o usuário clique no texto para focar/marcar a caixinha.

Principais Inputs e Campos:

  • type="text", "email", "password" → Campos de texto clássicos.
  • type="radio" → Permite escolher apenas UMA opção do grupo.
  • type="checkbox" → Caixinha de seleção (pode marcar várias).
  • <select> e <option> → Cria um menu suspenso (dropdown).
  • <textarea> → Caixa grande para textos longos.
                            HTML
<form>
  <label for="nome">Nome:</label>
  <input type="text" id="nome" placeholder="Seu nome">

  <label for="curso">Curso:</label>
  <select id="curso">

    <option value="html">HTML</option>
    <option value="css">CSS</option>
  </select>

  <label>
    <input type="checkbox"> Aceito os termos
  </label>
</form>

No passado, os sites eram feitos usando apenas blocos genéricos chamados <div>. O HTML5 introduziu as Tags Semânticas: elas fazem a mesma coisa que a div visualmente, mas seus nomes explicam qual o propósito do conteúdo. Isso é essencial para o Google ranquear seu site.

  • <header>: Cabeçalho do site ou de uma seção.
  • <nav>: Menu principal de navegação.
  • <main>: O conteúdo principal e exclusivo da página.
  • <section>: Uma seção genérica de agrupamento temático.
  • <article>: Conteúdo independente (como um post de blog ou notícia).
  • <footer>: Rodapé do site.
                            HTML
<body>
  <header>
    <h1>Meu Blog</h1>
    <nav><a href="#">Início</a></nav>
  </header>
  
  <main>
    <article>
      <h2>Notícia de Hoje</h2>
      <p>O HTML semântico salva a web!</p>
    </article>
  </main>

  <footer>Todos os direitos reservados.</footer>
</body>

O SEO (Otimização para Mecanismos de Busca) determina se seu site vai aparecer na primeira página do Google ou não. Muito disso é configurado nas Meta Tags, que ficam escondidas dentro da tag <head>.

  • <meta charset="UTF-8">: Garante que os acentos (ç, ã, é) apareçam corretamente.
  • <meta name="viewport"...>: Diz ao navegador que o site deve se adaptar a telas de celular (design responsivo).
  • <meta name="description" content="...">: A descrição curta que aparece embaixo do link do site no Google.
                            HTML
<head>
  <!-- Codificação e Responsividade -->
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  
  <!-- Título e Descrição para o Google -->
  <title>Curso de HTML - Code with Sam</title>
  <meta name="description" content="Aprenda HTML do zero ao avançado de forma fácil.">
</head>

O código não é escrito apenas para máquinas, mas também para outros programadores lerem e para pessoas com deficiências navegarem. Seguir as boas práticas diferencia um amador de um profissional.

  • Identação (Recuo): Use espaços ou "Tab" para empurrar o código para frente dentro das tags. Isso mostra claramente "quem está dentro de quem".
  • Comentários: Use <!-- texto --> para deixar recados no código. O navegador ignora os comentários, eles não aparecem na tela do usuário.
  • Acessibilidade (a11y): Sempre mantenha contraste alto de cores, textos descritivos em links (evite "clique aqui"), e nunca pule a hierarquia (não pule do H1 direto para o H3).
                            HTML
<!-- Jeito RUIM (Sem identação) -->
<div><p>Texto</p></div>

<!-- Jeito PROFISSIONAL -->
<div>
  <!-- Este é o parágrafo principal -->
  <p>Texto bem formatado e fácil de ler.</p>
</div>
